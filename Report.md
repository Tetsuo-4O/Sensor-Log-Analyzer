# **Report on Sensor Log Analyzer**

After picking the sensor log analyzer as our topic, the team decided it would be analyzing flight and GPS logs from UAS (unmanned aerial systems)/Drones. To achieve this, we needed to start with a menu function and decide what metrics we wanted to use and display to the user. After deliberating, we came up with five main functions that would be used in our menu function. They are functions that would allow the user of the program to access the hardware status, the event log, the trip log, and a custom log where they can choose which metrics to be displayed. Knowing what functions needed to be made, the work was divided between the team members with the goal of getting the menu done first, then integrating each function as they were finished. Following this plan, we were able to complete the program. 

When testing the code, a couple unforseen issues came up that were difficult to overcome. To start, our program was not dynamic when it came to file types, and it was easiest to only allow .csv files. When an incorrect file type was used, the code often led to a segmentation fault, which was fixed with an error message and then prompted the user to input the correct file path. The next issue to arise during testing was with the Custom Data Menu, where we needed to factor in certain edge cases. It was necessary to change the code to allow for putting in the same number multiple times in the custom menu and having a message when no selection was chosen. Lastly, we had an issue with getting the kml file to generate in the program directory when the trip log was selected. This was a case of adjusting formatting since kml files use “< >” as their brackets.

By fixing the countless errors in the code which were revealed from rigorous testing, we managed to learn valuable lessons that can be applied to our next projects. Some lessons we learned from fixing our errors during testing were the importance of file types and the syntax for those file types. Being able to only use .csv files is not the most user-friendly for a program, and it would have been nice to have a dynamic parser that could read multiple file types related to UASs. Along with this, knowing the correct syntax to use with certain file types showed to be more important than we thought. A major problem with generating the kml file was solved by a simple check of the brackets used. The process of working on a code like this, where errors are likely, then fixing them is valuable insight for the future.
